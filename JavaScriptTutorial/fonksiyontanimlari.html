JavaScript Fonksiyon Tanımları

JavaScript işlevleri, function anahtar sözcüğü ile tanımlanır.

Bir işlev bildirimi veya bir işlev ifadesi kullanabilirsiniz.

İşlev Beyanları

function functionName (parametreler) {

// yürütülecek kod

}

Bildirilen işlevler hemen yürütülmez. "Daha sonra kullanılmak üzere kaydedilirler" ve çağrıldıklarında (çağrıldıklarında) daha sonra çalıştırılırlar.

function myFunction (a, b) {

return * b;

}

Noktalı virgüller yürütülebilir JavaScript ifadelerini ayırmak için kullanılır.

İşlev bildirimi yürütülebilir bir ifade olmadığından, noktalı virgülle bitirmek yaygın değildir.

İşlev İfadeleri

Bir JavaScript işlevi bir ifade kullanılarak da tanımlanabilir.

Bir işlev ifadesi bir değişkende saklanabilir:

var x = function (a, b) {;

return * b

}

Bir işlev ifadesi bir değişkende saklandıktan sonra, değişken bir işlev olarak kullanılabilir:

var x = function (a, b) {;

return * b

}

var z = x (4, 3);

Yukarıdaki işlev aslında anonim bir işlevdir (adsız bir işlev).

Değişkenlerde saklanan fonksiyonların fonksiyon isimlerine ihtiyacı yoktur. Değişken adı kullanılarak her zaman çağrılır (çağrılır). Yukarıdaki işlev, noktalı virgülle biter, çünkü yürütülebilir bir ifadenin bir parçasıdır.

Function () Yapıcı Metodu

Önceki örneklerde gördüğünüz gibi, JavaScript işlevleri function anahtar sözcüğü ile tanımlanır.

İşlevler ayrıca Function () adlı yerleşik bir JavaScript işlev yapıcısıyla da tanımlanabilir.

var myFunction = new Function ("a", "b", "dönüş * b");

var x = myFunction (4, 3);

Aslında işlev yapıcısını kullanmak zorunda değilsiniz. Yukarıdaki örnek yazmakla aynıdır:

var myFunction = işlev (a, b) {dönüş * b};

var x = myFunction (4, 3);

Çoğu zaman, JavaScript'te new anahtar kelimesini kullanmaktan kaçınabilirsiniz.

Fonksiyon Kaldırma

Kaldırma, JavaScript'in bildirimleri geçerli kapsamın üstüne taşıma varsayılan davranışıdır.

Kaldırma, değişken bildirimler ve işlev bildirimleri için geçerlidir.

Bu nedenle, JavaScript işlevleri bildirilmeden önce çağrılabilir:

myFunction (5);

funciton myFunction (y) {

return y * y;

}

Bir ifade kullanılarak tanımlanan işlevler kaldırılmaz.

Kendini Çağırma İşlevleri

İşlev ifadeleri "kendi kendini çağıran" yapılabilir.

Kendini çağıran bir ifade çağrılmadan otomatik olarak çağrılır (başlatılır).

İfadeyi () izliyorsa işlev ifadeleri otomatik olarak yürütülür.

Bir işlev bildirimini kendiniz çağıramazsınız.

Bir işlev ifadesi olduğunu belirtmek için işlevin etrafına parantez eklemeniz gerekir:

(function () {

var x = "Merhaba !!"; // kendimi çağıracağım

}) ();

Yukarıdaki işlev aslında anonim kendi kendini çağıran bir işlevdir (adsız işlev).

İşlevler Değer Olarak Kullanılabilir

JavaScript işlevleri değer olarak kullanılabilir:

function myFunction (a, b) {

return * b;

}

var x = myFunction (4, 3);

JavaScript işlevleri ifadelerde kullanılabilir:

function myFunction (a, b)

return * b;

}

var x = myFunction (4, 3) * 2;

İşlevler Nesnelerdir

JavaScript'teki typeof operatörü işlevler için "function" döndürür. Ancak, JavaScript işlevleri en iyi nesne olarak tanımlanabilir.

JavaScript işlevlerinin hem özellikleri hem de yöntemleri vardır.

Argümanlar

Length özelliği, işlev çağrıldığında alınan bağımsız değişkenlerin sayısını döndürür:

function myFunction (a, b) {

return arguments.length;

}

ToString () yöntemi işlevi bir dize olarak döndürür:

function myFunction (a, b) {

return * b;

}

var txt = myFunction.toString ();

Bir nesnenin özelliği olarak tanımlanan bir işleve, nesneye yöntem denir.

Yeni nesneler oluşturmak için tasarlanmış bir işleve nesne yapıcısı denir.

Arrow İşlevleri

Arrow işlevleri, işlev ifadeleri yazmak için kısa bir sözdizimi sağlar.

Function anahtar sözcüğüne, return anahtar sözcüğüne ve süslü parantezlere ihtiyacınız yoktur.

// ES5

var x = function (x, y) {

return x * y;

}

// ES6

const x = (x, y) => x * y;

Arrow fonksiyonlarının kendilerine ait bir yönü yoktur. Nesne yöntemlerini tanımlamak için uygun değildirler.

Arrow fonksiyonları kaldırılmaz. Kullanılmadan önce tanımlanmalıdırlar.

Const kullanmak var kullanmaktan daha güvenlidir, çünkü bir işlev ifadesi her zaman sabit değerdir.

Return anahtar sözcüğünü ve süslü ayraçları yalnızca işlev tek bir deyimse atlayabilirsiniz.

Bu nedenle, onları her zaman tutmak iyi bir alışkanlık olabilir:

const x = (x, y) => {return x * y};
