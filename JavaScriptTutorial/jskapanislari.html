JavaScript Kapanışları

JavaScript değişkenleri yerel veya global kapsama ait olabilir.

Global değişkenler kapanışlarla yerel (özel) yapılabilir.

Global Değişkenler

Bir işlev, işlevin içinde tanımlanan tüm değişkenlere şu şekilde erişebilir:

function myFunction () {

var a = 4;

return a * a;

}

Ancak bir işlev, işlev dışında tanımlanan değişkenlere de şu şekilde erişebilir:

var a = 4;

function myFunction () {

return a * a;

}

Son örnekte, a global bir değişkendir.

Bir web sayfasında, global değişkenler pencere nesnesine aittir.

Genel değişkenler sayfadaki (ve penceredeki) tüm komut dosyaları tarafından kullanılabilir (ve değiştirilebilir).

İlk örnekte, a yerel bir değişkendir.

Bir yerel değişken sadece tanımlandığı fonksiyonun içinde kullanılabilir. Diğer işlevlerden ve diğer komut dosyası kodlarından gizlenir.

Aynı ada sahip genel ve yerel değişkenler farklı değişkenlerdir.

Birini değiştirmek diğerini değiştirmez.

Bir bildirim anahtar kelimesi olmadan oluşturulan değişkenler (var, let veya const), bir işlev içinde oluşturulmuş olsalar bile her zaman globaldir.

Değişken Ömrü

Genel değişkenler, başka bir sayfaya gittiğinizde veya pencereyi kapattığınızda olduğu gibi sayfa atılana kadar yaşar.

Yerel değişkenlerin kısa ömürleri vardır. Bunlar işlev çağrıldığında oluşturulur ve işlev tamamlandığında silinir.

Bir Karşı İkilem

Bir şeyi saymak için bir değişken kullanmak istediğinizi ve bu sayacın tüm işlevler için kullanılabilir olmasını istediğinizi varsayalım.

Genel bir değişken ve sayacı artırmak için bir işlev kullanabilirsiniz:

// Sayacı başlat

var sayaç = 0;


// Sayacı arttırma işlevi

function add () {

sayaç + = 1;

}


// 3 kez çağrı ekle ()

add();

add();

add();

// Sayaç şimdi 3 olmalı

Yukarıdaki çözümde bir sorun var: Sayfadaki herhangi bir kod, add () öğesini çağırmadan sayacı değiştirebilir.

Sayaç add () işlevinde yerel olmalıdır, diğer kodların değiştirilmesini önlemek için:

// Sayacı başlat

var sayaç = 0;


// Sayacı arttırma işlevi

function add () {

var sayac = 0;

sayaç + = 1;

}


// 3 kez çağrı ekle ()

add();

add();

add();


// Sayaç şimdi 3 olmalı. Ama 0

İşe yaramadı çünkü yerel sayaç yerine global sayacı görüntülüyoruz.

Genel sayacı kaldırabilir ve işlevin geri döndürmesine izin vererek yerel sayaca erişebiliriz:

// Sayacı arttırma işlevi

function add () {

var sayaç = 0;

sayaç + = 1;

return sayaç;

}


// 3 kez çağrı ekle ()

add();

add();

add();


// Sayaç şimdi 3 olmalı. Ama 1.

İşlevi her çağırdığımızda yerel sayacı sıfırladığımız için çalışmadı.

Bir JavaScript iç işlevi bunu çözebilir.

JavaScript İç İçe İşlevler

Tüm işlevlerin global kapsama erişimi vardır.

Aslında, JavaScript'te, tüm işlevlerin kendilerinin "üstündeki" kapsama erişimi vardır.

JavaScript iç içe işlevleri destekler. Yuvalanmış işlevler, "üstte" olan kapsama erişebilir.

Bu örnekte iç işlev plus (), üst işlevdeki sayaç değişkenine erişime sahiptir:

function add () {

var sayaç = 0;

function add () {sayaç + = 1;}

add();

return sayaç;

}

Dışarıdan plus () işlevine erişebilseydik, bu karşı ikilemi çözebilirdi.

Ayrıca counter = 0'ı yalnızca bir kez yürütmenin bir yolunu bulmamız gerekir.

Bir kapanmaya ihtiyacımız var.

var add = (function () {

var sayaç = 0;

return function () {counter += 1; return counter}

})();


add();

// sayaç şimdi 3

Açıklanan Örnek

Add değişkeni, kendini çağırma işlevinin dönüş değeri olarak atanır.

Kendini çağırma işlevi yalnızca bir kez çalışır. Sayacı sıfıra (0) ayarlar ve bir işlev ifadesi döndürür. Bu şekilde ekleme işlevi olur. "Harika" kısmı, ana kapsamdaki sayaca erişebilmesidir.

Buna JavaScript kapatma adı verilir. Bir fonksiyonun "özel" değişkenlere sahip olmasını mümkün kılar.

Sayaç, anonim işlevin kapsamı tarafından korunur ve yalnızca add işlevi kullanılarak değiştirilebilir.

Kapatma, üst işlev kapatıldıktan sonra bile üst kapsama erişime sahip bir işlevdir.
